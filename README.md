# 🐞 Bug Tracking System

The Bug Tracking System is a modern web-based application developed to streamline the process of reporting, tracking, managing, and resolving software bugs in agile development environments. Built using Python Flask, SQLite, HTML, CSS, and JavaScript, the system provides a centralized platform where Admins, Developers, and Testers can collaborate efficiently throughout the software development lifecycle.

The application features secure login and registration with role-based access control, ensuring that each user gets personalized access and functionality based on their role. Developers can view only assigned bugs, while administrators can monitor the complete system through a centralized dashboard.

Users can create detailed bug reports by entering information such as bug title, description, severity, priority, assigned developer, and deadlines through a clean and responsive bug reporting interface. The system helps teams organize issues efficiently and prioritize critical bugs faster.

The project also includes an advanced bug dashboard where users can view all reported bugs, monitor deadlines, identify overdue tasks, update bug statuses, and access detailed bug information. Bugs move through multiple lifecycle stages including Open, In Progress, and Closed, allowing real-time progress tracking.

One of the major highlights of the system is its interactive analytics dashboard that displays live statistics such as total bugs, open issues, resolved bugs, and developer-wise bug distribution. This helps administrators analyze workload and monitor project progress more effectively.

The platform maintains a complete comment and activity history for every bug, creating a transparent audit trail for communication between team members. Each update, status change, and discussion is stored for better collaboration and tracking.

Designed with a professional dark-themed UI, responsive layouts, animated components, and developer-friendly navigation, the project focuses strongly on usability, scalability, and modern user experience.

Unlike traditional bug management methods using spreadsheets or emails, the Bug Tracking System provides a more efficient, organized, and scalable workflow that improves collaboration, reduces issue resolution time, and enhances overall software quality!
